□ Click through all 7 steps

□ git init && git add . && git commit -m "Initial"
□ Push to GitHub
□ Go to vercel.com → Import your repo
□ Add environment variables
□ Click "Deploy" → ✅ Live!

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

❌ "Cannot find module"
   → Run: npm install (again)

❌ 3D robot not visible
   → Check: WebGL support (open DevTools → Console)
   → Fallback: Form still works, just no 3D

❌ Voice not working
   → Check: Speaker is not muted
   → Check: Browser has microphone permission
   → Try: Different browser (Chrome best)

❌ Forms not saving
   → Check: .env.local has correct URLs/keys
   → Verify: Supabase project is active
   → Test: Run query in Supabase SQL editor

❌ Deployment fails on Vercel
   → Check: All env vars in Vercel Settings
   → Verify: Push code to GitHub first
   → Try: Vercel rebuild (Settings → Deployments)

KEY DECISIONS MADE
═══════════════════════════════════════════════════════════════════════════

✅ Vercel + Supabase (vs Firebase/AWS)
   Why: Best ROI, easiest to learn, free tier sufficient
   Alternative: See ALTERNATIVES.md for other options

✅ React Three Fiber (vs native Three.js)
   Why: React-friendly, easier to maintain, less code
   Benefit: Full access to Three.js features when needed

✅ Zustand (vs Redux/Context)
   Why: Lightweight, perfect for this app, minimal boilerplate
   Benefit: State updates are super fast

✅ Tailwind CSS (vs styled-components)
   Why: Faster development, great defaults, excellent docs
   Benefit: Responsive design works instantly
